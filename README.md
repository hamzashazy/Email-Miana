# Miana Email Classification System

LLM-powered classifier that categorizes real estate offer reply emails into 5 categories using GPT-4o-mini with a domain-tuned prompt and structured JSON output. Processes 513 emails in ~90 seconds via async batching. The prompt is lean (~600 tokens) with edge-case disambiguation rules, and the architecture is provider-agnostic (swappable between OpenAI, Groq, or Gemini by changing the base URL).

## Categories

| Category | Description |
|---|---|
| **rejection** | Seller/agent clearly declined the offer, bounces/delivery failures |
| **counter_offer** | Seller/agent responds with a specific dollar amount or alternative terms |
| **interested** | Positive engagement — offer accepted, will present to seller, scheduling next steps |
| **replies** | Neutral acknowledgements, attachment issues, proof of funds requests, clarifying questions |
| **needs_review** | Ambiguous emails that require human review |

## Evaluation Results

Tested on 513 labeled emails (full dataset, zero API failures):

| Category | Precision | Recall | F1 | Support |
|---|---|---|---|---|
| rejection | 76.3% | 84.5% | 80.2% | 252 |
| counter_offer | 40.3% | 93.5% | 56.3% | 31 |
| interested | 22.0% | 61.1% | 32.4% | 18 |
| replies | 67.3% | 39.5% | 49.8% | 167 |
| needs_review | 71.4% | 22.2% | 33.9% | 45 |
| **Overall** | | | **64.1% accuracy** | **513** |

**Note on accuracy:** The labeled dataset was generated by keyword matching, which has inconsistent labels — ~70 emails labeled `replies` or `needs_review` are actually rejections, counter offers, or interested (e.g., "seller counters at $67K" labeled as `replies`). Accounting for dataset label noise, real-world accuracy is estimated at **~80%**.

## Setup

```bash
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your API key
```

## Usage

### Classify a batch of emails from xlsx

```bash
python classify.py --input email-classification-report.xlsx --output results.xlsx
```

Produces a color-coded xlsx with LLM classifications, confidence scores, and reasoning per email.

### Classify a single email

```bash
python classify.py --single --subject "Re: Cash Offer for 123 Main St" --body "No thanks, not interested."
```

### Evaluate against labeled data

```bash
python evaluate.py                          # Full evaluation (513 emails)
python evaluate.py --sample 50              # Quick sample
python evaluate.py --output eval-results.json  # Save detailed results
```

## Configuration

Environment variables (`.env`):

| Variable | Default | Description |
|---|---|---|
| `OPENAI_API_KEY` | — | OpenAI API key (required) |
| `OPENAI_MODEL` | `gpt-4o-mini` | Model to use |

To switch providers, update `BASE_URL` in `classifier/config.py`:
- OpenAI: `https://api.openai.com/v1`
- Groq: `https://api.groq.com/openai/v1`
- Gemini: `https://generativelanguage.googleapis.com/v1beta/openai/`
